## 23:12 - SQLのLIKE演算子

LIKE演算子は文字列のパターンマッチングに使用する。

**基本構文**
```sql
SELECT * FROM table WHERE column LIKE 'pattern';
```

**ワイルドカード**
- `%` : 0文字以上の任意の文字列
- `_` : 任意の1文字

**使用例**
```sql
-- "A"で始まる
WHERE name LIKE 'A%'

-- "son"で終わる
WHERE name LIKE '%son'

-- "at"を含む
WHERE name LIKE '%at%'

-- 2文字目が"a"
WHERE name LIKE '_a%'
```

**注意点**
- 大文字・小文字の区別はDBMSによって異なる（MySQLはデフォルト区別なし、PostgreSQLは区別あり）
- `%` や `_` 自体を検索したい場合はエスケープが必要（`ESCAPE`句を使用）
- インデックスが効きにくい場合がある（前方一致 `'A%'` はインデックス有効、後方一致 `'%A'` は無効）

## 23:13 - BETWEEN演算子とESCAPE

**BETWEEN演算子**

範囲検索に使用する。数値・日付・文字列に対応。

```sql
-- 基本構文
WHERE column BETWEEN value1 AND value2
-- 以下と同等
WHERE column >= value1 AND column <= value2
```

```sql
-- 数値の範囲
WHERE price BETWEEN 100 AND 500

-- 日付の範囲
WHERE created_at BETWEEN '2024-01-01' AND '2024-12-31'

-- 否定
WHERE price NOT BETWEEN 100 AND 500
```

**注意点**
- 両端の値を含む（inclusive）
- `NOT BETWEEN` で範囲外を検索できる

---

**ESCAPE句（LIKEと組み合わせて使用）**

`%` や `_` をワイルドカードではなくリテラルとして検索する際に使用。

```sql
-- "$" をエスケープ文字として指定
WHERE name LIKE '100$%' ESCAPE '$'
-- → "100%" という文字列を検索

WHERE code LIKE 'A$_01' ESCAPE '$'
-- → "A_01" という文字列を検索
```

**ポイント**
- `ESCAPE '文字'` で任意のエスケープ文字を指定できる
- エスケープ文字の直後の `%` や `_` がリテラルとして扱われる

## 23:14 - IN演算子

複数の値のいずれかに一致するかを検索する。

**基本構文**
```sql
WHERE column IN (value1, value2, value3, ...)
```

**使用例**
```sql
-- 複数の値を指定
WHERE status IN ('active', 'pending', 'review')

-- 否定
WHERE status NOT IN ('deleted', 'banned')

-- 数値
WHERE id IN (1, 2, 3, 10)
```

**サブクエリとの組み合わせ**
```sql
-- サブクエリの結果セットと照合
WHERE department_id IN (
    SELECT id FROM departments WHERE location = 'Tokyo'
)
```

**注意点**
- `OR` の連続と同等だが、`IN` の方が可読性が高い
- `NULL` は `IN` リストに含めても一致しない（NULLの比較は `IS NULL` を使う）
- `NOT IN` にサブクエリを使う場合、サブクエリ結果に `NULL` が含まれると全行が除外されるため注意

## 23:19 - ANY・ALL・AND・OR

**AND / OR**

複数条件の組み合わせ。

```sql
-- AND: すべての条件を満たす
WHERE age >= 20 AND status = 'active'

-- OR: いずれかの条件を満たす
WHERE status = 'active' OR status = 'pending'

-- 組み合わせ時はカッコで優先順位を明示
WHERE (status = 'active' OR status = 'pending') AND age >= 20
```

---

**ANY / SOME**

サブクエリの結果のいずれかと比較が真なら一致。`SOME` は `ANY` の別名。

```sql
-- サブクエリの結果のいずれかより大きい
WHERE price > ANY (SELECT price FROM products WHERE category = 'A')
-- → category='A' の最小値より大きければ一致

-- IN と同等の書き方
WHERE id = ANY (SELECT id FROM active_users)
-- → WHERE id IN (...) と同じ
```

---

**ALL**

サブクエリの結果のすべてと比較が真なら一致。

```sql
-- サブクエリの全結果より大きい
WHERE price > ALL (SELECT price FROM products WHERE category = 'A')
-- → category='A' の最大値より大きければ一致

-- NOT IN と同等の書き方
WHERE id <> ALL (SELECT id FROM banned_users)
-- → WHERE id NOT IN (...) と同じ
```

**ANY vs ALL まとめ**
| 演算子 | 意味 | 同等の表現 |
|--------|------|------------|
| `= ANY` | いずれかに一致 | `IN` |
| `<> ALL` | すべてに不一致 | `NOT IN` |
| `> ANY` | 最小値より大きい | `> MIN(...)` |
| `> ALL` | 最大値より大きい | `> MAX(...)` |

**注意点**
- `ALL` でサブクエリが空の場合、常に `TRUE` になる
- `ANY` でサブクエリが空の場合、常に `FALSE` になる
- サブクエリに `NULL` が含まれる場合の挙動に注意

## 23:49 - SELECTの修飾子・DISTINCT・ORDER BY・OFFSET-FETCH・UNION

**DISTINCT**

重複行を除去して取得する。

```sql
SELECT DISTINCT department FROM employees;

-- 複数列の場合、組み合わせが重複している行を除去
SELECT DISTINCT department, job_title FROM employees;
```

パフォーマンス注意：
- ソートや一時テーブルが発生するためコストが高い
- 重複が発生しない設計ならそもそも `DISTINCT` 不要
- `GROUP BY` で代替できる場合もある

---

**ORDER BY**

結果セットを並び替える。

```sql
-- 昇順（デフォルト）
SELECT * FROM employees ORDER BY salary;

-- 降順
SELECT * FROM employees ORDER BY salary DESC;

-- 複数列でソート
SELECT * FROM employees ORDER BY department ASC, salary DESC;

-- 列番号でも指定可（非推奨）
SELECT name, salary FROM employees ORDER BY 2 DESC;
```

パフォーマンス注意：
- インデックスがある列でのソートは高速
- インデックスがない列のソートは全件スキャン後にソート（`filesort`）
- `LIMIT` と組み合わせる場合もインデックスが有効なら効率的

---

**OFFSET / FETCH（ページネーション）**

```sql
-- SQL標準構文（SQL Server, PostgreSQL, Oracle 12c+）
SELECT * FROM employees
ORDER BY id
OFFSET 20 ROWS FETCH NEXT 10 ROWS ONLY;
-- 21〜30件目を取得

-- MySQL / PostgreSQL の LIMIT-OFFSET
SELECT * FROM employees
ORDER BY id
LIMIT 10 OFFSET 20;
```

パフォーマンス注意：
- `OFFSET` が大きくなるほど遅い（先頭から全件スキャンしてスキップ）
- 大量データのページネーションには「カーソル方式」が有効
  ```sql
  -- 前ページの最後のIDを条件にする（OFFSET不要）
  WHERE id > 1000 ORDER BY id LIMIT 10
  ```

---

**UNION / UNION ALL**

複数の SELECT 結果を縦に結合する。

```sql
-- UNION: 重複行を除去
SELECT name FROM employees
UNION
SELECT name FROM contractors;

-- UNION ALL: 重複行を保持（高速）
SELECT name FROM employees
UNION ALL
SELECT name FROM contractors;
```

パフォーマンス注意：
- `UNION` は重複除去のためソートが発生 → コスト高
- 重複が不要なら必ず `UNION ALL` を使う
- 列数・データ型を揃える必要がある

---

**SELECT修飾子まとめ（実行順序）**

```
FROM → WHERE → GROUP BY → HAVING → SELECT → DISTINCT → ORDER BY → OFFSET/FETCH
```

書く順序と実行順序が異なる点に注意。`WHERE` は `GROUP BY` より先に実行されるため、集計結果への条件は `HAVING` を使う。
