## 22:42 - SQLのCASE式

条件分岐を行う式。`SELECT`・`WHERE`・`ORDER BY` など様々な場所で使える。

**単純CASE式（値を比較）**
```sql
SELECT name,
    CASE status
        WHEN 'active'  THEN '有効'
        WHEN 'pending' THEN '保留'
        WHEN 'deleted' THEN '削除済'
        ELSE 'その他'
    END AS status_label
FROM users;
```

**検索CASE式（条件式で評価）**
```sql
SELECT name, salary,
    CASE
        WHEN salary >= 800000 THEN '高'
        WHEN salary >= 500000 THEN '中'
        ELSE '低'
    END AS salary_rank
FROM employees;
```

**ORDER BY での活用**
```sql
-- 特定の値を先頭に並べる
SELECT * FROM tasks
ORDER BY
    CASE status
        WHEN 'urgent' THEN 1
        WHEN 'normal' THEN 2
        ELSE 3
    END;
```

**集計との組み合わせ（条件付きカウント）**
```sql
SELECT
    COUNT(CASE WHEN status = 'active'  THEN 1 END) AS active_count,
    COUNT(CASE WHEN status = 'pending' THEN 1 END) AS pending_count
FROM users;

-- SUM でも同様
SELECT
    SUM(CASE WHEN gender = 'M' THEN salary ELSE 0 END) AS male_total
FROM employees;
```

**注意点**
- CASE式は「式」なので値を返す（文ではない）
- `ELSE` を省略すると一致しない場合は `NULL` になる
- 上から順に評価され、最初に一致した `WHEN` で終了（短絡評価）
- `NULL` との比較は `WHEN NULL THEN` ではなく `WHEN col IS NULL THEN` を使う（検索CASE式で）

## 22:48 - LENGTH関数・ユーザー定義関数・ストアドプロシージャ

**LENGTH関数（文字列長）**

```sql
-- バイト数を返す（MySQLのデフォルト）
SELECT LENGTH('hello');      -- 5
SELECT LENGTH('あいう');     -- 9（UTF-8では3バイト×3）

-- 文字数を返す
SELECT CHAR_LENGTH('あいう'); -- 3（MySQL）
SELECT LENGTH('あいう');      -- 3（PostgreSQL は文字数）
```

DBMSによって挙動が異なるため注意。

---

**ユーザー定義関数（UDF: User Defined Function）**

値を返す再利用可能な関数。SELECT の中で使える。

```sql
-- MySQL の例
CREATE FUNCTION get_tax(price DECIMAL(10,2))
RETURNS DECIMAL(10,2)
DETERMINISTIC
BEGIN
    RETURN price * 1.10;
END;

-- 呼び出し
SELECT name, get_tax(price) AS price_with_tax FROM products;
```

```sql
-- PostgreSQL の例
CREATE OR REPLACE FUNCTION get_tax(price NUMERIC)
RETURNS NUMERIC AS $$
BEGIN
    RETURN price * 1.10;
END;
$$ LANGUAGE plpgsql;
```

特徴：
- `SELECT` 内で使える（式として扱える）
- 値を返す（`RETURNS` が必須）
- 副作用（INSERT/UPDATE）を持たせないのが原則

---

**ストアドプロシージャ**

複数のSQL文をまとめた手続き。値を返さず（または OUT パラメータで返す）、処理をまとめるために使う。

```sql
-- MySQL の例
CREATE PROCEDURE update_salary(
    IN emp_id INT,
    IN raise_amount DECIMAL(10,2)
)
BEGIN
    UPDATE employees
    SET salary = salary + raise_amount
    WHERE id = emp_id;
END;

-- 呼び出し
CALL update_salary(1, 50000);
```

```sql
-- OUT パラメータで値を返す例
CREATE PROCEDURE get_employee_count(OUT cnt INT)
BEGIN
    SELECT COUNT(*) INTO cnt FROM employees;
END;

CALL get_employee_count(@result);
SELECT @result;
```

**UDF vs ストアドプロシージャ**
| 比較項目 | UDF | ストアドプロシージャ |
|----------|-----|----------------------|
| 呼び出し方 | `SELECT func()` | `CALL proc()` |
| 戻り値 | 必ず1つ返す | OUT パラメータで返す |
| SELECT内使用 | 可 | 不可 |
| 副作用（更新など） | 原則なし | あり |

**注意点**
- ストアドプロシージャはDBに依存するためポータビリティが低い
- デバッグが難しいためアプリ側ロジックで代替できる場合はそちらを推奨
- `DETERMINISTIC`（同じ入力で同じ出力）の指定はパフォーマンスに影響

## 22:49 - 空白除去関数（TRIM・LTRIM・RTRIM）

**TRIM**

両端の空白（または指定文字）を除去する。

```sql
SELECT TRIM('  hello  ');       -- 'hello'
SELECT TRIM('  hello  world  '); -- 'hello  world'（中間の空白は除去しない）

-- 特定の文字を除去（SQL標準）
SELECT TRIM(LEADING  'x' FROM 'xxxhelloxxx'); -- 'helloxxx'
SELECT TRIM(TRAILING 'x' FROM 'xxxhelloxxx'); -- 'xxxhello'
SELECT TRIM(BOTH     'x' FROM 'xxxhelloxxx'); -- 'hello'
```

**LTRIM / RTRIM**

```sql
SELECT LTRIM('  hello  ');  -- 'hello  '（左の空白を除去）
SELECT RTRIM('  hello  ');  -- '  hello'（右の空白を除去）
```

**DBMS別の対応**
| 関数 | MySQL | PostgreSQL | SQL Server |
|------|-------|------------|------------|
| `TRIM` | ○ | ○ | ○（SQL Server 2017+） |
| `LTRIM` | ○ | ○ | ○ |
| `RTRIM` | ○ | ○ | ○ |
| `TRIM(LEADING ...)` | ○ | ○ | △ |

**注意点**
- 全角スペース（`　`）は除去されない場合がある（DBMSや文字コードによる）
- 中間の空白を除去したい場合は `REPLACE(col, ' ', '')` を使う
- データ入力時の前後空白は INSERT 前にアプリ側で除去するのが望ましい

## 22:52 - 集計関数

複数行をまとめて1つの値を返す関数。`GROUP BY` と組み合わせてよく使う。

**主な集計関数**

```sql
SELECT
    COUNT(*)           AS 総行数,        -- NULL含む全行数
    COUNT(salary)      AS 給与有り件数,  -- NULL除く
    COUNT(DISTINCT dept) AS 部署数,      -- 重複除去してカウント
    SUM(salary)        AS 給与合計,
    AVG(salary)        AS 給与平均,
    MAX(salary)        AS 最高給与,
    MIN(salary)        AS 最低給与
FROM employees;
```

**GROUP BY との組み合わせ**

```sql
SELECT department, COUNT(*) AS 人数, AVG(salary) AS 平均給与
FROM employees
GROUP BY department;
```

**HAVING（集計後の絞り込み）**

```sql
-- 平均給与が50万以上の部署のみ
SELECT department, AVG(salary) AS avg_sal
FROM employees
GROUP BY department
HAVING AVG(salary) >= 500000;
```

`WHERE` は集計前、`HAVING` は集計後のフィルタ。

**NULLの扱い**
- `COUNT(*)` → NULL含む全行を数える
- `COUNT(列名)` → NULLを除いてカウント
- `SUM` / `AVG` / `MAX` / `MIN` → NULLを無視して計算
- 全行がNULLの場合、`SUM`/`AVG`/`MAX`/`MIN` は NULL を返す

**パフォーマンス注意点**
- `COUNT(DISTINCT col)` はコストが高い（ソートや一時テーブルが発生）
- `GROUP BY` する列にインデックスがあると高速化できる
- 大量データの集計はバッチ処理や集計テーブルの活用を検討する
