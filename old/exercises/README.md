## 🚀 使い方

### 1. 問題を読む
各Javaファイルの冒頭にある詳細なコメントを読んで、要件を理解してください。

### 2. クラスを実装する
各ファイルの `TODO` コメントに従って、必要なクラスを実装してください。

```java
// TODO: ここから下にクラスを実装してください
```

### 3. テストする
`main` メソッドでテストコードを書いて、動作を確認してください。

```bash

javac old/exercises/Exercise01_Vehicle.java

```

### 4. フォーマッタを適用（任意）
実装後、コードフォーマッタを適用できます。

```bash
./gradlew spotlessApply
```

## 📖 参考資料

### OOPの基本概念
- **カプセル化**: データと操作を1つのクラスにまとめ、外部からの直接アクセスを制限
- **継承**: 既存のクラスを基に新しいクラスを作成
- **ポリモーフィズム**: 同じインターフェースで異なる実装を扱う
- **抽象化**: 共通の特徴を抽出してクラスやインターフェースを定義

### Javaの重要キーワード
- `abstract`: 抽象クラス・抽象メソッドの定義
- `interface`: インターフェースの定義
- `extends`: クラスの継承
- `implements`: インターフェースの実装
- `@Override`: メソッドのオーバーライド
- `super`: 親クラスのメンバーへのアクセス
- `private`: クラス内からのみアクセス可能
- `public`: どこからでもアクセス可能
- `final`: 変更不可
