# ヘルスチェックエンドポイント

## 概要
`GET /api/health` エンドポイントを追加し、アプリケーションの稼働状態を確認できるようにする。

## 背景・目的
監視ツールやロードバランサーがアプリケーションの死活監視を行えるようにするため。

## 機能要件
- `GET /api/health` でアプリケーションの状態を返す
- ステータス、バージョン情報、現在時刻を含むレスポンスを返す
- 正常時は HTTP 200 を返す

## API仕様
- エンドポイント: `GET /api/health`
- リクエスト: なし
- レスポンス:
  ```json
  {
    "status": "UP",
    "version": "0.0.1-SNAPSHOT",
    "timestamp": "2026-03-31T12:00:00"
  }
  ```

## データモデル変更
なし

## 受け入れ条件
- [ ] `GET /api/health` が HTTP 200 を返す
- [ ] レスポンスに `status: "UP"` が含まれる
- [ ] レスポンスに `timestamp` が含まれる
- [ ] テストが追加されている
